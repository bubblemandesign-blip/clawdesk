name: Release ClawDesk
on:
  push:
    tags: ['v*.*.*']
  workflow_dispatch:

jobs:

  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v3
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - uses: dtolnay/rust-toolchain@stable
      - name: Install frontend dependencies
        run: pnpm install
      - name: Build Tauri app
        run: pnpm tauri build
      - name: Upload Windows installers
        uses: softprops/action-gh-release@v2
        with:
          files: |
            src-tauri/target/release/bundle/msi/*.msi
            src-tauri/target/release/bundle/nsis/*.exe
          token: ${{ secrets.GITHUB_TOKEN }}

  build-mac:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v3
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - name: Add Mac targets (Intel + Apple Silicon)
        run: |
          rustup target add aarch64-apple-darwin
          rustup target add x86_64-apple-darwin
      - name: Install frontend dependencies
        run: pnpm install
      - name: Build universal Mac app
        run: pnpm tauri build --target universal-apple-darwin
      - name: Upload Mac installer
        uses: softprops/action-gh-release@v2
        with:
          files: src-tauri/target/universal-apple-darwin/release/bundle/dmg/*.dmg
          token: ${{ secrets.GITHUB_TOKEN }}

  build-linux:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v3
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: '22' }
      - name: Install Linux dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y \
            libgtk-3-dev \
            libwebkit2gtk-4.1-dev \
            libayatana-appindicator3-dev \
            librsvg2-dev \
            patchelf
      - uses: dtolnay/rust-toolchain@stable
      - name: Install frontend dependencies
        run: pnpm install
      - name: Build Tauri app
        run: pnpm tauri build
      - name: Upload Linux AppImage
        uses: softprops/action-gh-release@v2
        with:
          files: src-tauri/target/release/bundle/appimage/*.AppImage
          token: ${{ secrets.GITHUB_TOKEN }}

STEP 3.6 — PLATFORM-SPECIFIC PROBLEMS TO PREVENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

These problems WILL happen on users' machines.
Handle all of them proactively before they reach us as bug reports.

── WINDOWS ──

Problem: "VCRUNTIME140.dll not found" or "MSVCP140.dll missing"
Cause: Microsoft Visual C++ Runtime not installed (common on fresh Windows)
Fix: In the NSIS/MSI installer, bundle the VC++ redistributable.
In tauri.conf.json under bundle.windows:
  "webviewInstallMode": { "type": "downloadBootstrapper" }
This makes the installer automatically install WebView2 and VC++ runtime.

Problem: WebView2 not installed (Windows 10 older versions)
Cause: Tauri uses WebView2 which ships with Windows 11 but may be missing on older Windows 10
Fix: "webviewInstallMode": { "type": "downloadBootstrapper" } (same as above)
This downloads and installs WebView2 silently if missing.

Problem: App installed but won't launch, no error shown
Cause: Usually WebView2 or VC++ runtime missing
Fix: Same as above — the bootstrapper handles it silently.

Problem: Windows Defender SmartScreen blocks .exe installer
Cause: No code signing certificate (costs $200+/year — skip for now)
Fix: Landing page clearly explains: "More info → Run anyway"
This is expected and normal for new unsigned apps.

Problem: App installs but can't find openclaw after reboot
Cause: PATH issues with local install
Fix: Always use absolute paths to openclaw binary — never bare "openclaw" command.
Path: [appDataDir]/engine/node_modules/.bin/openclaw.cmd (Windows)

── MAC ──

Problem: "ClawDesk is damaged and cannot be opened"
Cause: macOS quarantine attribute added by Safari/Chrome during download
Fix: Landing page instructions: xattr -cr /Applications/ClawDesk.app
This removes the quarantine flag.

Problem: "ClawDesk cannot be opened because developer cannot be verified"
Cause: No Apple Developer notarization ($99/year — skip for now)
Fix: Landing page instructions: Right-click → Open → Open
This bypasses Gatekeeper for unnotarized apps.

Problem: App works on Intel Mac but not Apple Silicon (M1/M2/M3)
Cause: Built for x86 only
Fix: ALWAYS use --target universal-apple-darwin in GitHub Actions.
Universal binary works on both. Never skip this flag.

Problem: App opens but WebView is blank white screen
Cause: macOS 14+ security changes to WebKit
Fix: In tauri.conf.json security.csp, ensure 'unsafe-eval' is included.
Already in our config above.

── LINUX ──

Problem: "AppImage cannot be executed" — permission denied
Cause: Downloaded without execute permission
Fix: Landing page instructions: chmod +x before running
This is standard for AppImages.

Problem: App launches but shows blank window
Cause: Missing WebKit/GTK libraries
Fix: Landing page instructions for apt-get install webkit2gtk
OR: User is on a minimal distro without desktop dependencies.

Problem: FUSE not available (needed for AppImage)
Cause: Older kernels or containers
Fix: Add to landing page Linux section:
  "If AppImage fails, try: ./ClawDesk.AppImage --no-sandbox"

── ALL PLATFORMS ──

Problem: OpenClaw gateway port 18789 already in use
Cause: Previous crash left process running, or another app uses this port
Fix: Already handled in code — kill port before starting.
Verify this code path exists and is tested.

Problem: First-time bootstrap fails silently at 0%
Cause: Firewall blocking npm registry (corporate networks, school networks)
User-visible fix: Show message "If setup is stuck, check your firewall or try a different network"
Show this after 60 seconds of no progress.

Problem: API key validation times out
Cause: User behind proxy or slow connection
Fix: Increase validation timeout to 15 seconds (not 5 or 10).
After timeout show: "Connection slow. Check your internet and try again."

Problem: App works first day, broken next day
Cause: Laptop sleep killed OpenClaw process
Fix: Already handled with onFocus restart.
Verify this is tested: close laptop, wait 5 minutes, open laptop, app should auto-recover.
